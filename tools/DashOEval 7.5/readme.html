Running DashO

A set of shell scripts and batch files are provided for running DashO:

  • dashogui / dashogui.bat - Runs the graphical user interface
  • dashocmd / dashocmd.bat - Runs the command line version
  • premark / premark.bat - Runs the command line version of the watermarking tool
  • dashorc / dashorc.bat - Resource files

Sample resource files are installed in DashO's home directory. You may alter these copies to affect the settings used by all users of DashO. You can also copy the files into a your home directory and customize the settings just for yourself. Please note that the shell script version should be renamed .dashorc when you place it in your home directory.

The resource files can customize these values:

  • javaOptions - These are options passed to the JVM.
  • memoryStart and memoryMax - The initial and maximum amount of memory that the JVM will allocate for DashO. Example: 192m for 192 megabytes.
  • options - Command line options passed to DashO.

PreEmptive Analytics

DashO supports PreEmptive Analytics. PreEmptive Analytics improves software quality, application adoption, and user satisfaction. DashO supports PA via annotations in your code, virtual annotations from a project file, direct API calls, or any combination of these. A JSP Taglib for PA messages is also provided. The Taglib is packaged in pa-taglib.jar. An example WAR file is provided: see the Read-me for details.

Ant Tasks

DashO includes tasks for use with Apache Ant. Please see the Ant Tasks file for installation and use instructions.

Eclipse Plugin

DashO includes a plugin for the Eclipse IDE that lets you create, edit and execute DashO project files from inside Eclipse. Please see the Eclipse Plugin file for installation and use instructions.

Gradle Plugin

DashO includes a plugin for Gradle that integrates DashO into the standard build process for java applications as well as the New Build System for Android. Please see the Gradle Plugin User Guide for additional information.

Map file API

DashO creates map files that record renaming information. An API is provided for working with these files. The API is packaged in mapfile.jar. Javadoc for the API is provided in the docs directory.
